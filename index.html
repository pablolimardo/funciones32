⚡ Hora 1: ¿Qué es el Paradigma Funcional?
🎮 Analogía: Las Funciones son Máquinas Puras
Imaginate que cada función es una máquina en una fábrica:

🔹 Le das una entrada (materia prima)
🔹 La máquina procesa (sin romper nada afuera)
🔹 Te devuelve una salida (producto terminado)

LO CLAVE: La máquina NO puede tocar otras máquinas, NO puede modificar el depósito de materiales, NO puede cambiar el clima de la fábrica. Solo hace SU trabajo con lo que le das.

👉 Esto se llama "función pura" — siempre da el mismo resultado con la misma entrada, sin efectos secundarios.

🤔 ¿Por Qué Importa el Paradigma Funcional?

Aprender programación funcional te hace mejor programador en CUALQUIER lenguaje. Aquí te muestro por qué:

🧪 Código más Testeable

Las funciones puras son FÁCILES de probar porque siempre dan el mismo resultado. No necesitás configurar hardware, limpiar variables globales, ni rezar. 

def celsius_a_fahrenheit(c):
    return c * 9/5 + 32

# Test: simple!
assert celsius_a_fahrenheit(0) == 32
assert celsius_a_fahrenheit(100) == 212
🔍 Bugs Menos Probables

Sin variables globales modificándose, hay MENOS lugares donde los bugs pueden esconderse. Tu código es predecible. 

# ❌ Bug común (imperativo)
contador = 0
def procesar():
    global contador  # Alguien modifica esto!
    contador += 1    # Sorpresa!

# ✅ Seguro (funcional)
def procesar(lista):
    return lista + [nuevo_elemento]
🔄 Código Reutilizable

Las funciones puras se pueden combinar y reutilizar fácilmente. Escribís menos código repetido. 

# Una función pura...
def doble(x): return x * 2

# ... se usa en cien contextos diferentes
numeros = [1, 2, 3, 4]
dobles = list(map(doble, numeros))
# Funciona siempre, sin configuración
⚡ Más Fácil de Debuggear

Si algo está mal, sabes exactamente dónde: la función pura. No hay sorpresas de efectos secundarios. 

# ✅ Fácil de debuggear
resultado = map(procesar, datos)
print(resultado)  # Ves exactamente qué pasó
🎯 Paralelización (Futuro)

Las funciones puras se pueden ejecutar en paralelo sin problemas. Las funciones con efectos? Pesadilla.

📚 Código Autodocumentado

map(convertir, temperaturas) es CLARO. Sabes exactamente qué pasa. Vs imperativos complejos.

💡 En el proyecto de casa domótica:

Cuando tenés 50 sensores leyendo temperaturas simultáneamente, las funciones puras son tu salvavidas. Sabés que cada función funciona exactamente como se supone, sin importar qué más está pasando en el sistema.

🔍 Repaso: ¿Qué es una Función?

Una función es un bloque de código reutilizable que:

  • Recibe entrada (parámetros o argumentos)
  • Realiza un trabajo específico
  • Retorna salida (resultado)
def nombre_funcion(parámetro1, parámetro2):
    # Aquí va el trabajo
    resultado = parámetro1 + parámetro2
    return resultado
Parte Explicación
def Palabra clave para definir una función
nombre_funcion El nombre que elegís para tu función (describe qué hace)
(parámetro1, parámetro2) Parámetros = lo que recibe la función (puede no haber ninguno)
return Devuelve el resultado de la función (salida)
💡 Ejemplo Cotidiano:

Una función es como una máquina expendedora:

  • Entrada: Puts money (parámetro)
  • Máquina procesa (código de la función)
  • Salida: Te da un producto (return)
📚 Conceptos Clave del Paradigma Funcional
1. Funciones Puras ✅
Una función pura siempre devuelve el mismo resultado con la misma entrada y NO modifica nada afuera.
Ejemplo Pura: 
def sumar(a, b):
    return a + b

resultado = sumar(3, 5)  # Siempre da 8
resultado = sumar(3, 5)  # Siempre da 8
# ✅ Pura porque: mismo input → siempre mismo output
Ejemplo IMPURA (con efectos): 
contador = 0
def incrementar():
    global contador
    contador += 1  # Modifica variable externa
    return contador

resultado = incrementar()  # Devuelve 1
resultado = incrementar()  # Devuelve 2 (diferente!)
# ❌ Impura porque: modifica el mundo exterior
2. Inmutabilidad 🔒
Los datos NO se modifican. Si necesitás cambiar algo, creás una copia nueva.
Estilo INMUTABLE (Funcional): 
temperaturas = [20, 21, 22]
# En lugar de modificar la lista...
nuevas_temps = temperaturas + [23]
# Creamos una NUEVA lista con el elemento agregado

print(temperaturas)   # [20, 21, 22] (sin cambios)
print(nuevas_temps)   # [20, 21, 22, 23] (nueva lista)
Estilo MUTABLE (Imperativo): 
temperaturas = [20, 21, 22]
# Modificamos la lista original
temperaturas.append(23)
print(temperaturas)   # [20, 21, 22, 23] (lista modificada)

¿Por qué importa? Si trabajás con copias, no hay sorpresas. Nadie puede modificar tus datos "por atrás".

3. Funciones como Ciudadanos de Primera Clase 👑
Las funciones se pueden pasar como argumentos, guardar en variables, y devolver desde otras funciones.
Ejemplo: Pasar una función como argumento 
def aplicar_operacion(a, b, operacion):
    # 'operacion' es una FUNCIÓN que pasamos como parámetro
    return operacion(a, b)

def sumar(x, y):
    return x + y

def multiplicar(x, y):
    return x * y

# Pasamos funciones como argumentos
print(aplicar_operacion(5, 3, sumar))        # 8
print(aplicar_operacion(5, 3, multiplicar))  # 15

¿Por qué es poderoso? Una misma función (aplicar_operacion) puede hacer cosas diferentes según la función que le pases.

4. Sin Efectos Secundarios 🎯
Una función NO debe modificar el mundo exterior (variables globales, archivos, pantalla).
PURA - Sin efectos: 
def procesar_temperatura(temp):
    # Solo calcula y retorna
    celsius = (temp - 32) * 5/9
    return celsius
    # ✅ No modifica variables globales
    # ✅ No imprime
    # ✅ No accede a archivos
IMPURA - Con efectos: 
archivo_log = open("log.txt", "a")

def procesar_temperatura(temp):
    celsius = (temp - 32) * 5/9
    print(f"Procesando: {celsius}")  # ❌ Efecto: imprime
    archivo_log.write(str(celsius))  # ❌ Efecto: modifica archivo
    return celsius

⚔️ Imperativo vs Funcional: La Batalla

¿Cuál es la diferencia fundamental?

Imperativo: "¿CÓMO hago algo?" → Describís paso a paso qué hacer
Funcional: "QUÉ quiero lograr?" → Describís el resultado deseado

Ejemplo: Calcular promedio de temperaturas

Querés calcular el promedio de: [22.5, 23.1, 21.8]

❌ IMPERATIVO (¿CÓMO?): 
temperaturas = [22.5, 23.1, 21.8]

# Paso 1: Crear variables
total = 0
cantidad = 0

# Paso 2: Iterar manualmente
for temp in temperaturas:
    total = total + temp
    cantidad = cantidad + 1

# Paso 3: Calcular resultado
promedio = total / cantidad

print(promedio)  # 22.466...
✅ FUNCIONAL (QUÉ?): 
from functools import reduce

temperaturas = [22.5, 23.1, 21.8]

# Una sola línea: "Quiero el promedio"
promedio = reduce(lambda a, t: a + t, temperaturas, 0) / len(temperaturas)

print(promedio)  # 22.466...
Por qué Funcional es mejor para este caso:
  • ❌ Imperativo: Necesitas rastrear total y cantidad en tu cabeza
  • ❌ Imperativo: Más líneas = más lugar para cometer errores
  • ✅ Funcional: Expresa claramente la intención: "Calcula el promedio"
  • ✅ Funcional: Menos variables, menos bugs
  • ✅ Funcional: Fácil de testear
🐍 Python Imperativo ❌ Verboso 
# Variable global (¡peligro!)
temperatura_total = 0
cantidad_mediciones = 0

def agregar_temperatura(temp):
    global temperatura_total, cantidad_mediciones
    temperatura_total += temp
    cantidad_mediciones += 1

def obtener_promedio():
    return temperatura_total / cantidad_mediciones

# Uso
agregar_temperatura(22.5)
agregar_temperatura(23.1)
agregar_temperatura(21.8)
promedio = obtener_promedio()

# Problemas:
# - Variables globales (confuso)
# - Estado mutable (cambios sorpresa)
# - Difícil de testear
# - Error propenso
🐍 Python Funcional ✅ Limpio 
# Sin variables globales

def calcular_promedio(temperaturas):
    total = sum(temperaturas)
    cantidad = len(temperaturas)
    return total / cantidad

# Uso
mediciones = [22.5, 23.1, 21.8]
promedio = calcular_promedio(mediciones)

# Ventajas:
# + Función pura (predecible)
# + No hay estado global
# + Fácil de testear
# + Inmutable (datos no cambian)
# + Reutilizable

🚀 Funciones de Orden Superior: El Siguiente Nivel

Una función de orden superior es una función que:

  • ✅ Recibe otra función como argumento, O
  • ✅ Devuelve una función como resultado
💡 Analogía: Máquina que configura máquinas

Una función de orden superior es como una máquina que crea o modifica otras máquinas. No hace el trabajo directamente, sino que orquesta a otras máquinas para hacerlo.

Ejemplo: Función que devuelve una función 
def crear_multiplicador(factor):
    """Esta función RETORNA otra función"""
    def multiplicar(x):
        return x * factor
    return multiplicar

# Creamos funciones personalizadas
duplicar = crear_multiplicador(2)
triplicar = crear_multiplicador(3)

print(duplicar(5))    # 10 (5 * 2)
print(triplicar(5))   # 15 (5 * 3)

# Misma estructura, diferentes comportamientos
# Eso es el poder de las funciones de orden superior

🗺️ Función: map()

map() aplica una función a cada elemento de una lista y devuelve una nueva lista con los resultados.

Sintaxis: 
resultado = list(map(funcion, lista))

• funcion: Una función que se aplicará a cada elemento
• lista: La lista con los elementos
• list(): Convertir el resultado a una lista (map devuelve un objeto especial)

Ejemplo: Convertir toda una lista de Celsius a Fahrenheit 
def celsius_a_fahrenheit(celsius):
    return (celsius * 9/5) + 32

# Lista de temperaturas en Celsius
temps_celsius = [18, 20, 22, 24, 26]

# Usar map() para convertir TODAS
temps_fahrenheit = list(map(celsius_a_fahrenheit, temps_celsius))

print(temps_celsius)      # [18, 20, 22, 24, 26]
print(temps_fahrenheit)   # [64.4, 68.0, 71.6, 75.2, 78.8]

# Paso a paso qué hace map():
# 1. Toma el primer elemento (18) → aplica celsius_a_fahrenheit(18) → 64.4
# 2. Toma el segundo elemento (20) → aplica celsius_a_fahrenheit(20) → 68.0
# 3. Toma el tercero (22) → aplica celsius_a_fahrenheit(22) → 71.6
# ... y así para todos los elementos
Comparación: map() vs bucle tradicional
Imperativo (bucle for): 
temps_fahrenheit = []
for celsius in temps_celsius:
    fahrenheit = celsius_a_fahrenheit(celsius)
    temps_fahrenheit.append(fahrenheit)

print(temps_fahrenheit)
Funcional (map): 
temps_fahrenheit = list(map(celsius_a_fahrenheit, temps_celsius))

print(temps_fahrenheit)

✅ El funcional es más corto y expresa mejor la intención: "Aplica esta función a toda la lista"

🔍 Función: filter()

filter() filtra elementos de una lista según una condición, devolviendo solo los que cumplen la condición.

Sintaxis: 
resultado = list(filter(funcion_que_retorna_bool, lista))

• funcion_que_retorna_bool: Una función que retorna True o False
• lista: La lista a filtrar
• Resultado: Una nueva lista SOLO con elementos donde la función retorna True

Ejemplo: Filtrar solo temperaturas confortables (18-24°C) 
def es_confortable(temperatura):
    """Retorna True si está en rango confortable"""
    return 18 <= temperatura <= 24

# Lista de temperaturas del día
temps = [15, 18, 20, 25, 22, 17, 23, 28]

# Filtrar solo las confortables
temps_ok = list(filter(es_confortable, temps))

print(temps)      # [15, 18, 20, 25, 22, 17, 23, 28]
print(temps_ok)   # [18, 20, 22, 23]

# Paso a paso qué hace filter():
# 1. ¿es_confortable(15)? → False → NO incluye 15
# 2. ¿es_confortable(18)? → True → INCLUYE 18
# 3. ¿es_confortable(20)? → True → INCLUYE 20
# 4. ¿es_confortable(25)? → False (> 24) → NO incluye 25
# ... y así para todos
Comparación: filter() vs bucle tradicional
Imperativo (bucle for): 
temps_ok = []
for temp in temps:
    if es_confortable(temp):
        temps_ok.append(temp)

print(temps_ok)
Funcional (filter): 
temps_ok = list(filter(es_confortable, temps))

print(temps_ok)

✅ El funcional es más limpio: "Filtra esta lista según esta condición"

📉 Función: reduce()

reduce() reduce una lista a UN ÚNICO valor aplicando una función acumuladora a todos los elementos.

Sintaxis: 
from functools import reduce
resultado = reduce(funcion_acumuladora, lista, valor_inicial)

• funcion_acumuladora: Función que recibe (acumulador, elemento_actual)
• lista: La lista a procesar
• valor_inicial: El valor de inicio (opcional)

Ejemplo: Calcular la suma de temperaturas 
from functools import reduce

def sumar(acumulado, temperatura):
    """Recibe el acumulado hasta ahora + elemento actual"""
    return acumulado + temperatura

temps = [20, 22, 21, 23, 25]

# Usar reduce para sumar todos
suma_total = reduce(sumar, temps, 0)  # 0 es el valor inicial

print(suma_total)  # 111

# Paso a paso qué hace reduce():
# 1. acumulado=0, temp=20 → sumar(0, 20) → 20
# 2. acumulado=20, temp=22 → sumar(20, 22) → 42
# 3. acumulado=42, temp=21 → sumar(42, 21) → 63
# 4. acumulado=63, temp=23 → sumar(63, 23) → 86
# 5. acumulado=86, temp=25 → sumar(86, 25) → 111
# Resultado final: 111
Otro ejemplo: Encontrar la temperatura máxima 
from functools import reduce

def encontrar_maximo(acumulado, temperatura):
    if temperatura > acumulado:
        return temperatura
    else:
        return acumulado

temps = [20, 22, 21, 23, 25, 19]
temperatura_maxima = reduce(encontrar_maximo, temps)

print(temperatura_maxima)  # 25

# Nota: Sin valor_inicial, usa el primer elemento (20)
# 1. acumulado=20 (primer elemento), temp=22 → 22
# 2. acumulado=22, temp=21 → 22
# 3. acumulado=22, temp=23 → 23
# ...
# Resultado: 25
Comparación: reduce() vs bucle tradicional
Imperativo (bucle for): 
suma = 0
for temp in temps:
    suma = suma + temp

print(suma)  # 111
Funcional (reduce): 
from functools import reduce
suma = reduce(lambda a, t: a + t, temps, 0)
print(suma)  # 111

✅ El funcional expresa claramente: "Reduce esta lista a un único valor sumando"

💡 Bonus: Lambda Functions (Funciones Anónimas)

A veces, la función que pasas a map/filter/reduce es muy pequeña. Podés usar lambda para escribirla en una línea:

Con función named: 
def sumar_5(x):
    return x + 5

resultado = list(map(sumar_5, [1, 2, 3]))
# [6, 7, 8]
Con lambda (más corto): 
resultado = list(map(lambda x: x + 5, [1, 2, 3]))
# [6, 7, 8]

# Sintaxis: lambda parámetros: expresión

📝 Función: list()

list() convierte un objeto iterable (como el resultado de map() o filter()) a una lista de Python que podés ver e iterar.

¿Por qué necesitamos list()?

map() y filter() devuelven objetos especiales llamados "iterables". Python los procesa bajo demanda (lazy evaluation = evaluación perezosa). Si querés ver todos los elementos a la vez, convertís a lista con list().

Ejemplo: Sin list() vs con list() 
temps = [20, 22, 21]

# Sin list() - No ves nada útil
resultado_sin_list = map(lambda x: x * 2, temps)
print(resultado_sin_list)  # <map object at 0x...> (objeto confuso)

# Con list() - Ves la lista claramente
resultado_con_list = list(map(lambda x: x * 2, temps))
print(resultado_con_list)  # [40, 44, 42] (mucho mejor!)
🎯 Resumen: Cuándo usás cada una
Función Propósito Devuelve
list() Convertir iterable a lista visible Una lista de Python
map() Transformar cada elemento Objeto map (úsalo con list())
filter() Seleccionar elementos que cumplen condición Objeto filter (úsalo con list())
reduce() Acumular en UN valor Un valor único (int, float, etc)
