❓ ¿Qué es una Función?
Una función es un bloque de código reutilizable que hace una tarea específica. En lugar de escribir el mismo código 100 veces, lo escribís una vez en una función y lo llamás cuando lo necesitás.
💡 Analogía: Una Máquina en una Fábrica

Imaginate una máquina que corta pan en rebanadas:

🔹 Le pasás un pan entero (entrada/parámetro)
🔹 La máquina procesa (ejecuta el código)
🔹 Te devuelve pan rebanado (salida/return)

Eso es una función: recibe entrada, procesa, devuelve salida.

🐍 Ejemplo: Función que suma dos números
def sumar(numero1, numero2):
    """Esta función suma dos números"""
    resultado = numero1 + numero2
    return resultado

# Ahora podés usarla cuando quieras
respuesta = sumar(5, 3)
print(respuesta)  # Imprime: 8
🧩 Partes de una Función
def

Palabra clave que dice "voy a crear una función"

def nombre_funcion()
Nombre de la Función

El nombre que elegís (debe describir qué hace)

def sumar()
Parámetros

Lo que recibe la función (puede no haber nada)

def sumar(a, b)
return

Lo que devuelve la función (el resultado)

return resultado
📋 Estructura Completa
def nombre_funcion(parámetro1, parámetro2):
    """Aquí va la explicación de qué hace"""
    
    # Aquí va el código que hace el trabajo
    resultado = parámetro1 + parámetro2
    
    # return: devuelve el resultado
    return resultado

# Usar la función
valor = nombre_funcion(5, 3)
print(valor)  # 8

📥 Parámetros y Return
📥 Parámetros: Lo que RECIBE la función

Los parámetros son variables que reciben valores cuando llamás la función. 

def sumar(a, b):  # 'a' y 'b' son parámetros
    return a + b

resultado = sumar(5, 3)  # 5 y 3 son los valores que pasan
# Dentro de la función: a=5, b=3
📤 Return: Lo que DEVUELVE la función

El return es lo que la función devuelve después de hacer su trabajo. 

def sumar(a, b):
    resultado = a + b
    return resultado  # Devuelve el resultado

valor = sumar(5, 3)
print(valor)  # 8 (el valor retornado por la función)
💡 Tip: Una función sin return

Si una función NO tiene return, devuelve None (nada).

def solo_imprime():
    print("Hola") (sin return)

valor = solo_imprime()
print(valor) # None

🔀 Modularización: Dividir el Código en Funciones
La modularización es dividir tu programa en funciones pequeñas, cada una haciendo una cosa. Es más fácil de entender, debuggear y reutilizar.
❌ SIN Modularización (todo mezclado): 
# Un solo bloque grande y confuso
temperatura = float(input("Temp: "))
if temperatura < 18:
    print("FRIO - Encender calefactor")
    archivo = open("log.txt", "a")
    archivo.write(f"Calefactor ON")
    archivo.close()
elif temperatura > 24:
    print("CALOR - Encender ventilador")
    archivo = open("log.txt", "a")
    archivo.write(f"Ventilador ON")
    archivo.close()
else:
    print("OK - Nada")
    
# Problema: código repetido, difícil de testear
✅ CON Modularización (funciones pequeñas): 
# Función 1: Clasificar
def clasificar_temperatura(temp):
    if temp < 18:
        return "FRIO"
    elif temp > 24:
        return "CALOR"
    else:
        return "OK"

# Función 2: Decidir acción
def decidir_accion(estado):
    if estado == "FRIO":
        return "Encender calefactor"
    elif estado == "CALOR":
        return "Encender ventilador"
    else:
        return "Nada"

# Programa principal (clara y corta)
temperatura = float(input("Temp: "))
estado = clasificar_temperatura(temperatura)
accion = decidir_accion(estado)
print(accion)

# Ventajas: Código limpio, fácil de testear, reutilizable
💡 Regla: Una función = Una cosa

Cada función debe hacer una sola cosa bien.
Si tu función hace 3 cosas → divídila en 3 funciones.

    if temp < 18:

    elif temp > 24:
        return "CALOR"
    else:

def activar_actuador(estado):
    if estado == "FRIO":
        return "CALEFACTOR: ON ✓"
    elif estado == "CALOR":
        return "VENTILADOR: ON ✓"
    else:
        return "SISTEMA: STANDBY"

# Función 4: Mostrar reporte

    print(f"Temperatura: {temp}°C")
    print(f"Estado: {estado}")

🔒 Scope: Variables Locales vs Globales
El scope es el "alcance" de una variable: dónde existe y dónde se puede usar.
📦 Variable Local

Existe solo dentro de la función. Se crea cuando entra a la función y desaparece cuando sale.

def ejemplo():
    x = 5  # Local
    return x

print(x)  # ERROR!
# 'x' NO existe aquí
🌍 Variable Global

Existe en todo el programa. Se crea fuera de funciones.

x = 5  # Global

def ejemplo():
    return x

print(x)  # OK - 5
# 'x' existe aquí también
⚠️ Evitá variables globales

Las variables globales complican el código. Es mejor usar parámetros y return.

Mejor: Pasar valores como parámetros → Función procesa → return resultado
