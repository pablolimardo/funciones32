⚡ Hora 1: ¿Qué es el Paradigma Funcional?
🎮 Analogía: Las Funciones son Máquinas Puras
Imaginate que cada función es una máquina en una fábrica:

🔹 Le das una entrada (materia prima)
🔹 La máquina procesa (sin romper nada afuera)
🔹 Te devuelve una salida (producto terminado)

LO CLAVE: La máquina NO puede tocar otras máquinas, NO puede modificar el depósito de materiales, NO puede cambiar el clima de la fábrica. Solo hace SU trabajo con lo que le das.

👉 Esto se llama "función pura" — siempre da el mismo resultado con la misma entrada, sin efectos secundarios.
En el paradigma funcional, programamos usando funciones puras que no modifican nada fuera de ellas. Esto hace que el código sea más predecible, testeable y fácil de mantener.
📚 Conceptos Clave del Paradigma Funcional
1. Funciones Puras
Siempre devuelven el mismo resultado con la misma entrada. No modifican variables externas.
2. Inmutabilidad
Los datos NO se modifican. Si necesitás cambiar algo, creás una copia nueva.
3. Funciones como Ciudadanos de Primera Clase
Las funciones se pueden pasar como argumentos, guardar en variables, y devolver desde otras funciones.
4. Sin Efectos Secundarios
Una función NO debe: modificar variables globales, imprimir en pantalla (salvo si es su propósito), cambiar archivos.

⚔️ Imperativo vs Funcional: La Batalla

🐍 Python Imperativo ❌ Con Efectos 
# Variable global (¡peligro!)
temperatura_total = 0
cantidad_mediciones = 0

def agregar_temperatura(temp):
    global temperatura_total, cantidad_mediciones
    temperatura_total += temp
    cantidad_mediciones += 1

def obtener_promedio():
    return temperatura_total / cantidad_mediciones

# Uso
agregar_temperatura(22.5)
agregar_temperatura(23.1)
agregar_temperatura(21.8)
promedio = obtener_promedio()

# Problema: Las funciones modifican 
# variables externas (efectos secundarios)
🐍 Python Funcional ✅ Pura 
# Sin variables globales

def calcular_promedio(temperaturas):
    total = sum(temperaturas)
    cantidad = len(temperaturas)
    return total / cantidad

# Uso
mediciones = [22.5, 23.1, 21.8]
promedio = calcular_promedio(mediciones)

# ✅ La función es pura:
# - No modifica nada externo
# - Siempre da el mismo resultado
# - Fácil de testear

void setup() {

void loop() {
  float t = dht.readTemperature();
  
  // Todo mezclado en loop()
  if (t < 18) {
    Serial.println("FRIO");
    digitalWrite(13, HIGH); // Calefactor
  } else if (t > 24) {
    Serial.println("CALOR");
    digitalWrite(12, HIGH); // Ventilador
  } else {
    Serial.println("OK");
    digitalWrite(13, LOW);
    digitalWrite(12, LOW);
  }
  
  delay(2000);
}
🤖 Arduino Funcional ✅ Modular 
#include "DHT.h"
#define DHTPIN 2
#define DHTTYPE DHT22
#define PIN_CALEFACTOR 13
#define PIN_VENTILADOR 12
DHT dht(DHTPIN, DHTTYPE);

// Funciones puras (solo calculan)
String clasificarTemp(float temp) {
  if (temp < 18) return "FRIO";
  if (temp > 24) return "CALOR";
  return "OK";
}

bool necesitaCalefaccion(float temp) {
  return temp < 18;
}

bool necesitaVentilacion(float temp) {
  return temp > 24;
}

// Función de efecto (actúa en hardware)
void controlarActuadores(float temp) {
  digitalWrite(PIN_CALEFACTOR, necesitaCalefaccion(temp));
  digitalWrite(PIN_VENTILADOR, necesitaVentilacion(temp));
}

void setup() {
  Serial.begin(9600);
  dht.begin();
  pinMode(PIN_CALEFACTOR, OUTPUT);
  pinMode(PIN_VENTILADOR, OUTPUT);
}

void loop() {
  float temp = dht.readTemperature();
  
  Serial.println(clasificarTemp(temp));
  controlarActuadores(temp);
  
  delay(2000);
}
✅ Ventajas del Código Funcional
  • Funciones pequeñas → Cada una hace UNA cosa
  • Fácil de testear → Podés probar clasificarTemp() sin hardware
  • Reutilizable → necesitaCalefaccion() se puede usar en otros lados
  • Legible → El loop() es super claro
